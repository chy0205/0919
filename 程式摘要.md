崇華十五期整合管理系統 - 技術摘要與維護指南
本文件旨在為未來的開發者提供系統架構的完整概觀，以便接手後續的修改與擴充。

1. 系統架構概念
本系統採用 Serverless (無伺服器) 架構：

前端 (Frontend)：單一 HTML 檔案 (index.html)，配合 Tailwind CSS、Vanilla JavaScript。
後端 (Backend)：Google Apps Script (GAS)，以 Web App 形式提供 RESTful API。
資料庫 (Database)：Google Sheets (試算表)。所有的使用者、點名、成績、作業紀錄皆儲存在指定的工作表中。
外部整合：
圖檔儲存：利用 GitHub API 將圖片存至 Repo，並透過 CDN 讀取。
定位功能：使用瀏覽器原生 Geolocation API。
2. 前端開發關鍵 (index.html)視圖切換：透過switchView(viewName) 函數在各個 <section> 之間切換顯示。
狀態管理：使用全域變數 currentUser 儲存登入資訊（並同步於 localStorage）。
非同步通訊：統一使用fetch(APPS_SCRIPT_URL, ...)與後端對接。
3. 後端開發關鍵 (GAS)進入點：doGet(e) 處理查詢類請求，doPost(e) 處理修改類請求（如註冊、繳交、改設定）。
動態表頭邏輯：
在 handleSubmitHomework 與 handleSubmitQuiz 中，程式會自動比對現有的欄位標題與傳入的題目名稱，若有新題目則自動在試算表右側新增欄位。
分頁策略：
測驗結果：按月份分頁（如 結果2026/03）。
作業內容：按主題分頁（如 作業_主題名稱）。
4. 維護與更新流程
修改前端：修改本地index.html -> 推送到 GitHub。
修改後端：在 Google Apps Script 編輯器修改 -> 部署 (New Version)。
環境變數：若更換試算表或後端 URL，需同步更新 index.html
 開頭的 APPS_SCRIPT_URL 與 SPREADSHEET_ID。
5. 常見問題排除
看不到紀錄：檢查 GAS getHistory 函數中的日期格式容錯處理，確保沒被無效列卡死。
上傳失敗：確認 GitHub Token 是否過期，或 Drive API 資料夾權限是否正確。
